SiardApi - Users' Manual - Usage

Usage

Version Handling in SiardApi

SiardApi is designed to replace SIARD Suite 1.0. Therefore it can read SIARD files in the SIARD Format 1.0, converting the meta data (e.g. of the BIT type) as needed. When an existing SIARD file is opened using SiardApi, closing it will have the following effect: If no meta data were changed, the SIARD file remains unchanged, i.e. both meta data and primary data are left exactly as they were. If, on the other hand, some meta data were modified, then the header folder in the ZIP64 file is changed to the new format according to the SIARD Format 2.1. However, the primary data will remain unchanged. If they were uncompressed (as in SIARD Format 1.0) they remain uncompressed, even though the meta data have been upgraded to version 2.1. A file in SIARD 1.0 format is thus upgraded to the SIARD 2.1 format but does not make use of the new permission to store the primary data in a compressed format. (Luckily the old BIT data were already stored like BINARY data are today.)

Only when a new SIARD file is created, the primary data are guaranteed to be written according to the new SIARD Format 2.1 with compressed primary data and possibly externalized large objects.

Documentation of SiardApi

The javadoc documentation of the API is distributed with SiardApi in the folder doc/javadoc of the distribution. In addition the test folder of the distribution contains sample source code using the SiardApi for reading an writing SIARD files. In addition this manual contains useful information about SiardApi.

Top of page

No GUI for SiardApi

SiardApi is distributed as a collection of JAR files without a graphical user interface (GUI). The sample code may indicate how to implement one. The project SiardGui implements a graphical user interface similar to SIARD Suite 1.0 using SiardApi for reading and writing SIARD files.

SiardApi can be used for reading and writing files in the SIARD Format 2.1 independently from an database. Therefore it is published as a separate stand-alone project.
